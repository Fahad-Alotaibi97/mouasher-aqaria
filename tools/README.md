# أداة استيراد البيانات — مؤشر العقارية

تحوّل ملفات CSV الرسمية (من [data.gov.sa](https://od.data.gov.sa) أو [الهيئة العامة للعقار — المؤشرات العقارية](https://rega.gov.sa/rega-services/platforms/real-estate-indicators/)) إلى كود SQL جاهز للّصق في Supabase.

**ليه بهذي الطريقة؟** الاستيراد عن طريق توليد SQL ولصقه في محرر Supabase آمن تماماً — ما يحتاج أي مفاتيح سرية، وأنت تشوف البيانات قبل ما تدخل.

## الخطوات

١. حمّل ملف CSV من المصدر الرسمي (مثلاً: متوسطات إيجارات أحياء الرياض).

٢. شغّل الأمر في الطرفية (Terminal) داخل مجلد المشروع:

```bash
# لمتوسطات الأحياء:
node tools/import-data.mjs neighborhoods المسار/للملف.csv

# للإعلانات:
node tools/import-data.mjs listings المسار/للملف.csv
```

٣. بيتولّد ملف SQL في مجلد `supabase/`. افتحه، انسخ محتواه، والصقه في:
   **Supabase → SQL Editor → New query → Run**.

## الأعمدة المدعومة

الأداة تتعرّف تلقائياً على رؤوس أعمدة عربية وإنجليزية شائعة:

- **الأحياء:** عمود للاسم (`الحي` / `name`) وعمود للمتوسط (`متوسط الإيجار` / `avg_rent`).
- **الإعلانات (إلزامية):** `العنوان`/`title`، `الحي`/`hood`، `النوع`/`type`، `السعر`/`advertised`.
- **اختيارية:** المساحة، الغرف، رخصة فال، `lat`، `lng`، الحالة.

لو كان الملف الرسمي بأعمدة مختلفة، أرسله لي وأضيف أسماء أعمدته للأداة فوراً.
