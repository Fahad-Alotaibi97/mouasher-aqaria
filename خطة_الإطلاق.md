# خطة إطلاق منصة "مؤشر العقارية"

ترتيب منطقي من الأساس للإطلاق. الرمز 👤 = تسويها أنت، 🤖 = أسويها أنا، 🤝 = نتعاون.

---

## المرحلة ١ — الأساس النظامي (شغلك أنت، يمشي بالتوازي مع التقني)
1. 👤 **السجل التجاري** — عبر المركز السعودي للأعمال (نشاط تقني/منصة عقارية).
2. 👤 **التحقق من متطلبات الهيئة العامة للعقار** — هل تشغيل منصة تعرض إيجارات وتتحقق من فال يحتاج ترخيص؟ (استشر مختصاً — حاسمة).
3. 👤 **التسجيل في معروف (maroof.sa)** — توثيق يعطي ثقة للعملاء.

## المرحلة ٢ — البنية الدائمة (السيرفر + الرابط)
4. 🤝 **النشر على Vercel** — يجعل المنصة تشتغل ٢٤/٧ بدون جهازك (هذا بديل localhost). أوجّهك خطوة بخطوة.
5. 🤝 **ضبط متغيّرات البيئة على Vercel** — رابط ومفتاح Supabase (لتشتغل البيانات على الموقع المنشور).
6. 👤 **حجز النطاق www.مؤشر.sa** — عبر SaudiNIC (يتطلب السجل التجاري غالباً).
7. 🤝 **ربط النطاق بـ Vercel + HTTPS** — يصير الرابط رسمي وآمن (قفل أخضر).
8. 🤖 **تحديث روابط Supabase Auth للنطاق الجديد** — عشان تسجيل الدخول يشتغل على الرابط الرسمي.

## المرحلة ٣ — قاعدة البيانات وحفظ بيانات العملاء
9. 👤 **تشغيل ملفات SQL في Supabase** — `schema.sql` + `update_2_fields_leads.sql` (تنشئ الجداول + جدول الرسائل).
10. 👤 **إنشاء Storage bucket باسم listings (Public)** — لتخزين صور الوحدات.
11. ✅ **حفظ بيانات العملاء** — جاهز: نموذج "اترك رسالة" يحفظ في جدول leads، ورسائل المكاتب تُحفظ. (تُقرأ من لوحة Supabase).

## المرحلة ٤ — حل بيانات الأسعار (سكني / إيجار / البيانات المفتوحة)
12. 🤝 **مصدر متوسطات الأحياء** — القرار: نعتمد **المؤشر الإيجاري الرسمي** (مبني على عقود إيجار، عبر سكني). نحمّل ملف البيانات ونستورده بالأداة الجاهزة (`tools/import-data.mjs`).
   - ملاحظة: سهيل منصة تجارية منافِسة وليست مصدر بيانات مفتوح — لا نسحب منها.
13. 🤖 **استيراد المتوسطات** — بمجرد ما تعطيني ملف البيانات، أحوّله SQL وتشغّله، فتصير الأسعار رسمية لكل أحياء الرياض.
14. 👤 **مصدر الإعلانات الحقيقية** — من المكاتب (يرفعونها عبر لوحة المكتب الجاهزة) أو إدخال أولي يدوي. (معضلة: تحتاج عرض قبل الطلب).

## المرحلة ٥ — ما قبل الإطلاق (مصداقية وقانون) — إلزامي
15. 🤝 **قرار "فال"** — التحقق حالياً وهمي. إما تفعيل حقيقي (يحتاج مصدر/واجهة رسمية) أو إزالة ادعاء "موثّق بفال". لا تطلق بادعاء غير حقيقي.
16. ✅ **سياسة الخصوصية + الشروط** — جاهزة (صياغة عامة؛ يُفضّل مراجعة محامٍ).
17. 🤖 **إزالة الأرقام الوهمية من لوحة المكتب** — الإحصاءات والاستفسارات الثابتة تُستبدل ببيانات حقيقية أو تُخفى.
18. 🤖 **ضبط الجوال (Mobile)** — أغلب الباحثين من الجوال.

## المرحلة ٦ — التحكم بالتكاليف (مهم قبل الإطلاق)
19. ✅ **لا مساعد ذكي** — أزلناه، فلا استهلاك توكنز/تكلفة.
20. 👤 **متابعة باقات Supabase/Vercel المجانية** — كافية للبداية؛ ننتبه للحدود مع نمو الاستخدام.

## المرحلة ٧ — الإطلاق
21. 🤝 **تجربة مغلقة** — رابط رسمي لعدد محدود (أصدقاء/مكاتب أولى) + جمع ملاحظات.
22. 🤝 **الإطلاق العام** — بعد تثبيت البيانات والمصداقية.

---

## الوضع الحالي (ما الجاهز فعلاً)
- ✅ الكود: بطاقات موحّدة + تفاصيل، نموذج رسالة، رفع وحدات بصور، خريطة، تسجيل دخول، صفحات قانونية.
- ⏳ ينقص للتشغيل الحقيقي: تشغيل ملفات SQL + إنشاء bucket + النشر على Vercel + بيانات الأسعار الرسمية + قرار فال.

## أسرع طريق لإطلاق تجريبي محترم (المسار القصير)
المرحلة ٢ (نشر) → المرحلة ٣ (SQL + bucket) → خطوة ١٣ (استيراد المتوسطات) → خطوة ١٥ (قرار فال) → خطوة ٢١ (تجربة مغلقة).
