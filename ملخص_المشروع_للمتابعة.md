# مؤشر العقارية — ملخص شامل للمتابعة

## نبذة عن المشروع
منصة ذكاء اصطناعي للإيجار السكني في الرياض. ميزتها الأساسية: **مؤشر "السعر العادل"** —
يقول للمستأجر فوراً إن كان سعر الإيجار (مرتفع / عادل / فرصة) مقارنةً بمتوسط الحي.
الجمهور: ٢٣–٤٥ سنة. الفلسفة: الحكم في أقل من ١٥ ثانية.

- **الإطار:** Next.js 16.2.6 (Turbopack) + React + TypeScript + Tailwind
- **قاعدة البيانات:** Supabase (مشروع: mouasher-aqaria)
  - Project URL: https://fytgsizewhjgxhxzezut.supabase.co
  - المفتاح العام (publishable) محفوظ في ملف .env.local
- **GitHub:** https://github.com/Fahad-Alotaibi97/mouasher-aqaria
- **الموقع المنشور (قديم، ما يحتوي تعديلاتنا):** https://mouasher-aqaria.vercel.app
- **مجلد المشروع:** C:\Users\Admin\OneDrive\سطح المكتب\mouasher-aqaria
- **التشغيل المحلي:** npm run dev ثم http://localhost:3000

## ما تم إنجازه ✅
1. تصحيح عنوان الموقع ووصفه (SEO) + lang=ar + dir=rtl.
2. تثبيت Supabase + ملفات الربط (client/server/config) + إعداد .env.local.
3. **قاعدة البيانات:** أُنشئت الجداول في Supabase (تم تشغيل SQL بنجاح):
   neighborhoods, profiles, offices, listings, leads — مع RLS ومتوسطات الأحياء.
4. **Storage bucket باسم "listings" (Public):** تم إنشاؤه لتخزين صور الوحدات.
5. تسجيل دخول بالرابط السحري (Magic Link) عبر Supabase Auth + صفحة /auth/callback.
   - أُضيف Redirect URL: http://localhost:3000/auth/callback في Supabase.
6. ربط الإعلانات والأحياء من قاعدة البيانات (مع رجوع آمن لبيانات افتراضية).
7. فصل المنطق في ملفات lib (types, fairPrice, useAppData, useAuth) — لإعادة الاستخدام بتطبيق جوال مستقبلاً.
8. أداة استيراد بيانات رسمية CSV→SQL في tools/import-data.mjs.
9. **استبدال المساعد الذكي بنموذج "اترك رسالة"** (اسم/جوال/رسالة) يُحفظ في جدول leads.
   - كود المساعد الأصلي محفوظ في: _محفوظات/المساعد-الذكي-route.ts.txt
10. **توحيد بطاقات الإعلانات** (صيغة واحدة بدون إيموجي): معلومات رئيسية على البطاقة
    (السعر/الحكم/السعر العادل/الغرف/المساحة/الحمامات) + نافذة تفاصيل عند الضغط.
11. **شاشة المكاتب لرفع الوحدات:** نموذج بحقول منظّمة + رفع صور إلى Storage + نشر فعلي.
12. نقل "اترك رسالة" لآخر الصفحة + صفحة سياسة خصوصية وشروط استخدام (تظهر عند الضغط بالفوتر).
13. تحويل خط الموقع إلى IBM Plex Sans Arabic + Tajawal.

## الخطوات القادمة (مرتّبة)

### فوراً — إصلاح التشغيل
1. حذف كاش .next (بعد إيقاف OneDrive أو نقل المشروع) → npm run dev → فتح localhost:3000.
2. اختبار: تسجيل الدخول، نموذج الرسالة، بطاقات الإعلان، رفع وحدة من لوحة المكتب.

### قبل الإطلاق العام (إلزامي)
3. **قرار رخصة فال:** التحقق حالياً وهمي (محاكاة) — إما تفعيل حقيقي أو إزالة ادعاء "موثّق بفال". (مصداقية + قانون)
4. **بيانات الأسعار الرسمية:** اعتماد "المؤشر الإيجاري" الرسمي (عبر سكني/إيجار) — تحميل ملف واستيراده بأداة tools/import-data.mjs. ملاحظة: سهيل منصة منافسة وليست مصدر بيانات مفتوح.
5. **إعلانات حقيقية:** من المكاتب (عبر اللوحة الجاهزة) أو إدخال أولي.
6. مراجعة الصفحات القانونية بواسطة مختص.

### النشر (لتشتغل المنصة ٢٤/٧ بدون الجهاز)
7. رفع التعديلات على GitHub (الأعمال الجديدة لم تُرفع بعد).
8. النشر على Vercel + ضبط متغيرات البيئة (Supabase URL + key).
9. النطاق www.مؤشر.sa عبر SaudiNIC (رسوم سنوية) + ربطه بـ Vercel + HTTPS.
10. تحديث Redirect URLs في Supabase للنطاق الجديد.

### المرحلة النظامية (بالتوازي — مسؤولية فهد)
11. السجل التجاري + التحقق من متطلبات الهيئة العامة للعقار + التسجيل في معروف.

### تحسينات مؤجلة
12. معالجة واجهة المكتب (الإحصاءات والاستفسارات حالياً أرقام ثابتة → ربطها بالحقيقي).
13. ضبط الجوال (Mobile-first).
14. لاحقاً: تطبيق جوال React Native يعيد استخدام نفس Supabase والمنطق.
